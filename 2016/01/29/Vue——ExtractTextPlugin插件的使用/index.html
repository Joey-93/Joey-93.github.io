Vue——ExtractTextPlugin插件的小坑

  • ExtractTextPlugin
  • extract-text-webpack-plugin
  • vue
  • webpack

点进来才让你看~~！

直接说问题

多entry前提下，ExtractTextPlugin要该配置才能正确输出对应entry的css文件?

网上的教程，绝大部分都是只针对单entry。

有一个不算完美的解决方法

写了这么久的vue，一直都是自己拼，觉得就算有人一起协作，应该也是懂vue的帅哥美女才对。
好吧，原因是这样的。

由于博主的书写样式能力着实很差，被项目经理吐槽了一番后，决定找个懂css、html的美工来拯救我的页面，可是美工估计vue都不知道，更别提webpack环境下的vue文件。
那只好这样，把美工懂的html、css文件扔给他，一切就解决了。

说干咱就干。

首先是上npm来一套。点这里

上述配置很简单，不过考虑到环境是在使用vue-loader，所以再到vue-loader的官方介绍上看看。点这里

也是挺简单，所以没想什么，直接就把代码copy到项目的webpack配置文件中。(当然也有一点点修改)。

按照预想，webpack应该会把vue中的style单独分离出来，全部都加载到style.css文件中。
结果:

一看style.css大小，好像有点不对劲，一打开…

总共就20行代码。样式写得再烂也不可能只有20行而已。况且上面的样式竟然是项目里面的一个vue组件中的style。
好吧，本来命名成style.css就有点不爽了(想自动根据entry去命名)。

解决方法


不懂webpack，只好google。

答案出来了…

首先是webpack官方的介绍，(英语不好，实在不想上去)



一看new ExtractTextPlugin("[name].css") 解决了我想根据entry自动去命名的需求。


再一看，好吧，原来是少了这一点，{allChunks:true}。

结果:

多个css文件出来了！

在多个entry中，需要添加这样一个options，这样webpack就会自动将每个entry中的所有样式都集中到对应的[name].css文件中。这样子，美工只需要修改对应的css文件就好了。


总结：

    1. webpack的文档确实有点凌乱，以至于一开始都没注意到上述两点，其实这也并不算坑，本身前端这段时期鱼龙混杂，别说英文教程了，中文教程估计也没几个能耐心看下去，都是直接上手各种新框架。vue-loader的文档中Extracting CSS File这块确实写得比较简略，就好比官网的异步组件介绍一样短。不过也不是什么问题，毕竟程序猿要自己动手动脑子，自己努力的东西，才是自己的。
    2. 前面所以说不是完美的解决办法，还有另外一点是我还没有理解，美工修改的只是编译之后的文件，就算修改正确了，对源文件也是没有任何帮助(总不可能我自己去对比并替换源文件中的style吧)，这点只能后面再看看有没有什么好的解决办法，貌似HtmlWebpackPlugin是一个方向。
